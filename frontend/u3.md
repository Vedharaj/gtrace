# Add Global Trace Playback Controls to Dashboard

## Objective

Add a **Global Trace Playback Toolbar** to the Dashboard page.

The toolbar controls execution replay of the loaded gem5 trace.

It should be positioned at the **top center**, immediately above the "5-Stage Pipeline CPU Datapath Visualizer" section (or above the primary execution visualization on the Dashboard).

These controls are global and synchronize every visualization in GTrace.

Changing the execution cycle updates:

- Datapath
- Pipeline
- Registers
- Memory
- Cache
- Timeline
- Editor
- Branch Predictor

The toolbar should remain visible while viewing the Dashboard.

# User Experience Goal

The playback toolbar should behave like a media player for CPU execution.

Users should feel they are replaying processor execution cycle-by-cycle.

The controls should synchronize every visualization in GTrace, allowing researchers to observe how instructions move through the datapath, registers, cache, memory, and pipeline during execution.

The toolbar should become the central interaction component for execution replay while maintaining the clean Scientific Minimalism design language.
